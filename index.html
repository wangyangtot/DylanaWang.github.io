Publications

User Data Can Tell Defaulters in P2P Lending.

Jackson J. Mi, Tianxiao Hu and Luke Deer, Conference on Data Science(ICDS'17)(Oral Presentation) & Annals of Data Science(AODS), 2018. view online

Online peer-to-peer (P2P) lending service is a new type of financial platforms that enables individuals borrow and lend money directly from one to another. In this paper, we analyzed the differences between credit levels and users’ distribution of CPLP to evaluate if the credit levels can truly reflect the borrowers’ credit. We used soft factors to establish a model that can ﬁnd borrowers who are likely to default. Further, we proposed some strategies to construct and improve the risk-control of P2P lending platforms.

Projects

SmartGomoku: A web version of the Gomoku game with AI agents.

Gomoku, also called Gobang or Five in a Row. It's usually played on a Go board. Players alternate turns placing a stone of their color on an empty intersection. The winner is the first player to form an unbroken chain of five stones horizontally, vertically, or diagonally.

We developed a web app to play Gomoku. Users can access the web page by computer or mobile phone. We employed weui when designing the user interface. The algorithm of agents and back end is completed using python and Flask framework.

We implemented six AI agents. Users can select one agent and play Gomoku with it or let two AI agents play with each other(see the demo left). The strategies of AI agents includes greedy algorithm, alpha-beta pruning and MCTS. In this project we also evaluated different evaluation functions and used numba package to accelerate the computing progress. To avoid losing advantages at the very beginning of the game, we also taught our agents some formulas.

This is the final course project for Introduction of Artificial Intelligence (DATA130008.01) @Fudan University. For further details, please visit repo here.

Glutton: a website for takeout foods.

Glutton means someone that never stops eating. As you might guess, it's designed for takeout foods. For users, they can create an account, manage profile, search for food and restaurant, place an order and view order history. For restaurant-owners, they can publish their restaurant info, change menu and view transaction history.

The front end is GitHub-like. For back end, Flask and SQLite were employed for fast development. This is the final course project for Introduction of Database Systems (COMP130010.03) @Fudan University. For further details, please visit repo here.
